                Reference

                Core concepts

                The ideas that underpin every project. Learn these and you'll understand what you're building.

                ETL
                ETL / ELT
                Extract, Transform, Load — the fundamental data movement pattern. ELT is the modern cloud variant: load raw first, transform inside the warehouse.
                DAG
                Orchestration
                Directed Acyclic Graphs define task dependencies. Airflow schedules and monitors pipelines. Everything runs on a schedule — or on a trigger.
                dbt
                Transformation layer
                dbt brings software engineering to SQL. Version control, tests, documentation, and modular SQL — all in one tool. Industry standard.
                ★★★
                Medallion architecture
                Bronze (raw) → Silver (cleaned) → Gold (business-ready). A layered warehouse pattern used by every serious data team.
                ≋≋
                Stream processing
                Kafka moves events in real time. Spark processes them with windowing and stateful aggregations. The difference between yesterday's data and now.
                ∆
                Delta Lake
                ACID transactions on top of data lakes. Time travel queries. Schema evolution. The technology that makes Databricks tick.
                ✓
                Data quality
                Tests that run in your pipeline and scream when data breaks. Great Expectations, dbt tests, data contracts. What separates production from prototype.
                {}
                Infrastructure as code
                Terraform defines your cloud infra in code. Reproducible, version-controlled, destroyable. Never click through a console again.
